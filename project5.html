Project 5

Multi-Zone Home Security System

نظام حماية منزلي متعدد المناطق باستخدام 8086 و 8255

← الرجوع للصفحة الرئيسية

1. شرح المشكلة وفهم فكرتها

في نظام حماية منزلي، قد يكون لدينا عدة أبواب ونوافذ وكل واحد منها عليه حساس مغناطيسي. إذا فتح أي باب أو نافذة، يجب أن يعرف النظام بسرعة أي منطقة تم اختراقها، ثم يشغل الإنذار.

لو تم توصيل كل حساس بدائرة مستقلة أو فحص كل حساس بطريقة منفصلة، سيزيد عدد الأسلاك ويصبح النظام أكثر تعقيداً. لذلك نستخدم 8255 حتى نقرأ 8 حساسات دفعة واحدة من خلال Port A.

فكرة المشروع ببساطة

يتم توصيل 8 حساسات أبواب ونوافذ على Port A. كل حساس يمثل Bit واحد. إذا كانت قيمة Port A تساوي 00H، فهذا يعني أن كل الأبواب والنوافذ مغلقة. وإذا تغيرت القيمة، يعرف البرنامج أن هناك اختراقاً، ثم يبحث عن البت الذي أصبح 1 ويعرض رقم المنطقة على Port C، ويشغل الإنذار من Port B.

2. شرح الحل باستخدام 8255

يتم برمجة Port A كمدخل Input لأنه يستقبل إشارات الحساسات. ويتم برمجة Port B كمخرج Output لتشغيل الصفارة ولمبة الإنذار وإشارة القفل. أما Port C فيتم استخدامه كمخرج لعرض رقم المنطقة التي تم فتحها.

البرنامج يقرأ Port A باستمرار. إذا كانت القيمة 00H، يبقى النظام في الحالة الآمنة. إذا كانت القيمة مختلفة عن الصفر، يقوم البرنامج بتخزين القيمة ثم يستدعي دالة FIND_ZONE لتحديد أول بت مفعّل، وبعدها يرسل رقم المنطقة إلى Port C.

بعد تحديد المنطقة، يدخل البرنامج في حلقة إنذار. في هذه الحلقة يتم تشغيل الصفارة ولمبة الإنذار ثم إطفاء الصفارة بشكل متكرر، بحيث يكون الإنذار متقطعاً. ويظل النظام في حالة الإنذار حتى تعود قراءة Port A إلى 00H.

3. Interface Map

Port A = Input PA0 = Door 1 Sensor
PA1 = Door 2 Sensor
PA2 = Window 1 Sensor
PA3 = Window 2 Sensor
PA4 = Back Door Sensor
PA5 = Garage Sensor
PA6 = Balcony Sensor
PA7 = Emergency Zone
Port B = Output PB0 = Siren / Buzzer
PB1 = Red Alarm LED
PB2 = Security Lock Signal
Port C = Output Displays Zone Number
01H = Zone 1
02H = Zone 2
...
08H = Zone 8

Control Word المستخدم هو 90H. معناه أن Port A مدخل Input، و Port B و Port C مخارج Output، وكلها تعمل في Mode 0.

4. الكود الكامل للمشكلة

ORG 100H

; -------------------------------
; 8255 Port Addresses
; -------------------------------
PORTA   EQU 300H
PORTB   EQU 301H
PORTC   EQU 302H
CTRL    EQU 303H

; -------------------------------
; Main Program
; -------------------------------
START:
    MOV DX, CTRL
    MOV AL, 90H
    OUT DX, AL

    ; System starts in secure state
    MOV DX, PORTB
    MOV AL, 00H
    OUT DX, AL

    MOV DX, PORTC
    MOV AL, 00H
    OUT DX, AL

MAIN_LOOP:
    ; Read all 8 security sensors
    MOV DX, PORTA
    IN  AL, DX

    ; If no sensor is active, stay secure
    CMP AL, 00H
    JE SECURE_STATE

    ; Save sensor pattern
    MOV BL, AL

    ; Find which zone is active
    CALL FIND_ZONE

    ; Display zone number on Port C
    MOV DX, PORTC
    MOV AL, CL
    OUT DX, AL

    JMP ALARM_LOOP

; -------------------------------
; Secure State
; -------------------------------
SECURE_STATE:
    ; Turn off siren, alarm LED, and lock signal
    MOV DX, PORTB
    MOV AL, 00H
    OUT DX, AL

    ; Clear zone display
    MOV DX, PORTC
    MOV AL, 00H
    OUT DX, AL

    JMP MAIN_LOOP

; -------------------------------
; Alarm State
; -------------------------------
ALARM_LOOP:
    ; Siren ON + Red LED ON + Lock signal ON
    MOV DX, PORTB
    MOV AL, 07H
    OUT DX, AL

    CALL SHORT_DELAY

    ; Siren OFF, keep Red LED and Lock signal ON
    MOV DX, PORTB
    MOV AL, 06H
    OUT DX, AL

    CALL SHORT_DELAY

    ; Read sensors again
    MOV DX, PORTA
    IN  AL, DX

    ; Stay in alarm while any sensor is active
    CMP AL, 00H
    JNE ALARM_LOOP

    JMP SECURE_STATE

; -------------------------------
; Find active zone number
; Input : BL = sensor pattern
; Output: CL = zone number
; -------------------------------
FIND_ZONE PROC
    PUSH AX

    MOV AL, BL
    MOV CL, 01H

CHECK_ZONE:
    TEST AL, 01H
    JNZ ZONE_FOUND

    SHR AL, 1
    INC CL

    CMP CL, 09H
    JNE CHECK_ZONE

    MOV CL, 00H

ZONE_FOUND:
    POP AX
    RET
FIND_ZONE ENDP

; -------------------------------
; Short Delay
; -------------------------------
SHORT_DELAY PROC
    PUSH CX

    MOV CX, 0FFFFH

D1:
    LOOP D1

    POP CX
    RET
SHORT_DELAY ENDP

END START

5. شرح الكود سطر بسطر

السطر الشرح
ORG 100H يحدد أن البرنامج من نوع COM ويبدأ من العنوان 100H.
PORTA EQU 300H يعطي اسم PORTA للعنوان 300H، وهو المنفذ المستخدم لقراءة حساسات الأبواب والنوافذ.
PORTB EQU 301H يعطي اسم PORTB للعنوان 301H، وهو المنفذ المستخدم لتشغيل الإنذار ولمبة الخطر والقفل.
PORTC EQU 302H يعطي اسم PORTC للعنوان 302H، وهو المنفذ المستخدم لعرض رقم المنطقة المخترقة.
CTRL EQU 303H يعطي اسم CTRL للعنوان 303H، وهو Control Register الخاص بإعداد 8255.
START: بداية البرنامج الرئيسية.
MOV DX, CTRL يضع عنوان Control Register داخل DX حتى نرسل له Control Word.
MOV AL, 90H يضع قيمة Control Word داخل AL. القيمة 90H تعني Port A مدخل و Port B و Port C مخارج.
OUT DX, AL يرسل 90H إلى Control Register، وبذلك يتم ضبط اتجاه المنافذ في 8255.
MOV DX, PORTB يضع عنوان Port B داخل DX حتى يتم إطفاء مخارج الإنذار في البداية.
MOV AL, 00H يضع صفر في AL. الصفر يعني أن كل مخارج Port B مطفأة.
OUT DX, AL يرسل 00H إلى Port B، فيكون الإنذار ولمبة الخطر والقفل مطفأة في البداية.
MOV DX, PORTC يضع عنوان Port C داخل DX حتى يتم تصفير شاشة أو مؤشر المنطقة.
MOV AL, 00H يضع صفر في AL من أجل مسح رقم المنطقة.
OUT DX, AL يرسل 00H إلى Port C، أي لا توجد منطقة مخترقة في البداية.
MAIN_LOOP: بداية حلقة المراقبة المستمرة لحساسات الحماية.
MOV DX, PORTA يضع عنوان Port A في DX حتى يقرأ الحساسات.
IN AL, DX يقرأ حالة Port A ويضعها في AL. كل بت في AL يمثل حساساً معيناً.
CMP AL, 00H يقارن قيمة AL بالصفر لمعرفة هل كل الحساسات مطفأة أم لا.
JE SECURE_STATE إذا كانت قيمة AL تساوي 00H، فهذا يعني أن البيت آمن، فيذهب إلى SECURE_STATE.
MOV BL, AL يحفظ نمط الحساسات داخل BL حتى يتم استخدامه في دالة تحديد المنطقة.
CALL FIND_ZONE يستدعي دالة FIND_ZONE لتحديد رقم أول منطقة فيها حساس مفعّل.
MOV DX, PORTC يحدد Port C لأننا سنعرض عليه رقم المنطقة.
MOV AL, CL ينقل رقم المنطقة من CL إلى AL حتى نرسله إلى Port C.
OUT DX, AL يرسل رقم المنطقة إلى Port C لعرض أي باب أو نافذة تم فتحها.
JMP ALARM_LOOP بعد تحديد المنطقة، ينتقل البرنامج إلى حلقة الإنذار.
SECURE_STATE: بداية الجزء الخاص بالحالة الآمنة.
MOV DX, PORTB يحدد Port B لإطفاء الإنذار ومؤشرات الخطر.
MOV AL, 00H يضع صفر في AL لإطفاء كل مخارج Port B.
OUT DX, AL يرسل 00H إلى Port B، فيتم إطفاء الصفارة واللمبة والقفل.
MOV DX, PORTC يحدد Port C لمسح رقم المنطقة.
MOV AL, 00H يضع صفر في AL لأن لا توجد منطقة مخترقة.
OUT DX, AL يرسل 00H إلى Port C، فيتم تصفير عرض المنطقة.
JMP MAIN_LOOP يرجع إلى حلقة المراقبة الرئيسية.
ALARM_LOOP: بداية حلقة الإنذار التي تعمل عند وجود اختراق.
MOV DX, PORTB يحدد Port B لأنه يتحكم بالصفارة ولمبة الإنذار والقفل.
MOV AL, 07H يضع 07H في AL. هذه القيمة تشغل PB0 و PB1 و PB2، أي الصفارة واللمبة والقفل.
OUT DX, AL يرسل 07H إلى Port B، فيتم تشغيل الإنذار الكامل.
CALL SHORT_DELAY ينفذ تأخيراً قصيراً والصفارة شغالة.
MOV DX, PORTB يحدد Port B مرة أخرى لتغيير حالة الصفارة.
MOV AL, 06H يضع 06H في AL. هذه القيمة تطفئ الصفارة PB0 وتبقي اللمبة والقفل شغالين PB1 و PB2.
OUT DX, AL يرسل 06H إلى Port B، فيصبح الإنذار متقطعاً لأن الصفارة تطفئ مؤقتاً.
CALL SHORT_DELAY ينفذ تأخيراً قصيراً والصفارة مطفأة.
MOV DX, PORTA يحدد Port A لقراءة الحساسات مرة ثانية.
IN AL, DX يقرأ حالة الحساسات مرة أخرى لمعرفة هل ما زال الاختراق موجوداً.
CMP AL, 00H يقارن قراءة الحساسات بالصفر.
JNE ALARM_LOOP إذا كانت القيمة ليست صفراً، فهذا يعني أن هناك حساساً لا يزال نشطاً، فيبقى داخل الإنذار.
JMP SECURE_STATE إذا عادت كل الحساسات للصفر، يرجع النظام للحالة الآمنة.
FIND_ZONE PROC بداية دالة تحديد رقم المنطقة النشطة.
PUSH AX يحفظ قيمة AX في Stack حتى لا تضيع أثناء تنفيذ الدالة.
MOV AL, BL ينقل نمط الحساسات من BL إلى AL حتى يتم فحصه داخل الدالة.
MOV CL, 01H يبدأ رقم المنطقة من 1، لأن أول حساس PA0 يمثل Zone 1.
CHECK_ZONE: بداية حلقة فحص البتات واحداً بعد الآخر.
TEST AL, 01H يفحص أقل بت في AL. إذا كان 1 فهذا يعني أن المنطقة الحالية هي المفتوحة.
JNZ ZONE_FOUND إذا كان البت المفحوص يساوي 1، يقفز إلى ZONE_FOUND لأن المنطقة تم تحديدها.
SHR AL, 1 يزحزح AL خانة واحدة لليمين حتى ينتقل لفحص البت التالي.
INC CL يزيد رقم المنطقة بمقدار 1 لأننا انتقلنا إلى الحساس التالي.
CMP CL, 09H يقارن CL مع 09H للتأكد أننا لم نتجاوز المناطق الثمانية.
JNE CHECK_ZONE إذا لم نصل إلى 9، يستمر في فحص البتات.
MOV CL, 00H إذا لم يجد أي بت مفعّل، يجعل رقم المنطقة 0 كحالة احتياطية.
ZONE_FOUND: هذا المكان يصل إليه البرنامج عندما يجد المنطقة النشطة.
POP AX يعيد قيمة AX القديمة من Stack.
RET يرجع من دالة FIND_ZONE إلى البرنامج الرئيسي.
FIND_ZONE ENDP نهاية دالة تحديد المنطقة.
SHORT_DELAY PROC بداية دالة التأخير القصير.
PUSH CX يحفظ قيمة CX في Stack قبل استخدامه في التأخير.
MOV CX, 0FFFFH يضع قيمة كبيرة في CX لصنع تأخير بسيط.
D1: بداية حلقة التأخير.
LOOP D1 ينقص CX ويكرر الحلقة حتى يصبح CX صفراً.
POP CX يعيد قيمة CX القديمة بعد انتهاء التأخير.
RET يرجع من دالة التأخير القصير.
SHORT_DELAY ENDP نهاية دالة التأخير القصير.
END START نهاية البرنامج، والتنفيذ يبدأ من START.

ملخص سريع للمناقشة

في هذا المشروع استخدمنا 8255 كواجهة لنظام حماية منزلي. Port A يعمل كمدخل لقراءة 8 حساسات أبواب ونوافذ، وPort B يعمل كمخرج لتشغيل الصفارة ولمبة الإنذار والقفل، وPort C يعمل كمخرج لعرض رقم المنطقة التي تم اختراقها. إذا كانت قراءة Port A تساوي 00H فالنظام آمن، وإذا تغيرت القيمة يقوم البرنامج بتحديد أول بت مفعّل باستخدام TEST و SHR، ثم يعرض رقم المنطقة ويشغل إنذاراً متقطعاً حتى تعود الحساسات إلى الوضع الطبيعي.
